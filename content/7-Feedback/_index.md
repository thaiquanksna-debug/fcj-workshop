# Feedback and Suggestions

## General Evaluation

### 1. Working Environment

The project was conducted in a well-structured and professional environment. Clear objectives and milestones helped maintain progress throughout the implementation process. The overall workflow encouraged both independent learning and practical problem-solving.

### 2. Guidance and Support

The guidance provided during the project was highly beneficial. Technical discussions and feedback sessions helped me better understand cloud architecture, infrastructure automation, and security best practices. Whenever challenges arose, support was available to help identify suitable solutions.

### 3. Relevance to My Field of Study

The project closely matched the knowledge and skills developed during my Information Technology studies. It provided opportunities to apply concepts from networking, cloud computing, operating systems, database systems, and cybersecurity in a practical environment.

### 4. Learning Experience

One of the most valuable aspects of this project was the opportunity to gain hands-on experience with modern cloud technologies and infrastructure management tools.

Key technologies and concepts learned include:

- Amazon Web Services (AWS)
- Terraform Infrastructure as Code
- Amazon VPC networking
- Amazon RDS PostgreSQL
- AWS Key Management Service (KMS)
- AWS CloudTrail logging
- AWS CloudWatch monitoring
- Open Policy Agent (OPA)
- Rego policy validation
- Secure cloud architecture design

The project also improved my troubleshooting, technical documentation, and infrastructure planning skills.

### 5. Communication and Collaboration

Although the project was largely implemented individually, communication remained an important part of the development process. Presenting architecture decisions, documenting implementation details, and discussing technical solutions helped strengthen my communication skills.

### 6. Project Organization

The project followed a clear roadmap with defined milestones. This structure made it easier to manage tasks, track progress, and verify that security and infrastructure requirements were successfully implemented.

---

## Additional Questions

### What was the most valuable thing you learned?

The most valuable lesson was understanding how security can be integrated into infrastructure from the beginning rather than being added later. Designing a private-by-default architecture provided practical experience in secure cloud deployment.

### What was the biggest challenge?

The biggest challenge was maintaining a fully private environment while ensuring that all components could still communicate and operate correctly. Careful configuration of networking, IAM permissions, encryption, and policy validation was required.

### Which skill improved the most?

Terraform and AWS architecture design showed the greatest improvement. I became more confident in managing infrastructure through code and implementing security-focused cloud solutions.

---

## Suggestions and Expectations

Overall, this project provided an excellent learning experience and exposed me to industry-relevant cloud engineering practices.

For future projects, additional activities such as architecture reviews, security assessments, and advanced cloud service integrations could further enhance learning opportunities.

I appreciate all the guidance and support received throughout the project and believe similar hands-on projects will continue to benefit future students.
